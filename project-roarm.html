Project Overview

To improve the control, accuracy, and usability of the RO-ARM m2 and m3 series robotic arms, I developed and currently maintain an open-source Python library. This project abstracts complex mathematical models and low-level serial communication into an intuitive Software Development Kit (SDK) designed for the robotics community, allowing for seamless integration into larger autonomous systems.

Installation

The library is published on PyPI and can be easily installed into any Python environment via pip. It is designed to be lightweight and highly responsive.

$pip install roarm-ik-solver

Quick Start Guide

Controlling the hardware is as simple as initializing the arm class and providing the desired Cartesian coordinates. The SDK handles the serial communication and hardware constraints automatically.

from roarm_sdk.roarm import m3Arm
import time

# Initialize the M3 Pro Arm via serial port
arm = m3Arm(port="/dev/ttyUSB0", baudrate=115200)

# Move the end-effector to X: 15, Y: 0, Z: 10 with a Phi angle of 0
arm.move_to_xyz(phi=0, x=15, y=0, z=10, wait=True)

# Draw a linear trajectory to a new point
arm.draw_line(phi=0, x1=15, y1=0, z1=10, x2=20, y2=10, z2=5, steps=30)

The Mathematics Behind the Movement

The core challenge of controlling a multi-axis robotic arm is translating desired 3D coordinates into specific rotational angles for each servo motor. I engineered a robust solver utilizing trigonometric laws.

First, we calculate the required base rotation to face the target coordinates in the $XY$ plane:

$$ \theta_{base} = \arctan2(Y, X) $$

To calculate the shoulder and elbow joint angles ($\theta_1$ and $\theta_2$), we project the 3-dimensional target onto a 2-dimensional planar frame and compute the physical distance $d$ to the target. Using the Law of Cosines alongside the known linkage lengths ($r_1$ and $r_2$), we solve for the inner elbow angle:

$$ \text{Inner Elbow} = \arccos\left(\frac{r_1^2 + r_2^2 - d^2}{2 \cdot r_1 \cdot r_2}\right) $$
$$ \theta_{elbow} = \pi - \text{Inner Elbow} $$

The SDK runs these calculations in real-time, verifying that the target distance $d$ does not exceed the maximum physical reach of the arm ($r_1 + r_2$), actively preventing self-collision and hardware damage.

Software & Python SDK Architecture

  • API Abstraction: Built a Python wrapper that translates the high-level IK solutions into low-level JSON/serial commands that the RoArm microcontroller natively understands.
  • Real-Time Execution: Optimized the Python code to run efficiently, ensuring low-latency communication between the host computer (e.g., NVIDIA Jetson or Raspberry Pi) and the robotic arm for real-time tracking tasks.
  • Open Source Community: Fully documented the codebase and published it to GitHub, providing installation instructions, dependencies, and example scripts for immediate use by hobbyists and researchers.

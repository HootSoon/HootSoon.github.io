Project Overview

During my NASA OSTEM Internship at the Johnson Space Center, my primary objective was to modernize the visual fidelity of aerospace example simulations. I achieved this by bridging NASA's core physics engine with industry-leading 3D rendering and modeling software.

The NASA Trick Framework

The Trick Simulation Environment is a powerful simulation architecture developed by NASA. It handles the heavy lifting of aerospace simulations—managing the rigorous physics calculations, math models, timing, and data logging required to simulate complex systems like orbital mechanics and rocket launches.

While Trick is incredibly robust at calculating the math, the original visual outputs for example simulations required an upgrade to ensure its modern use in the new space age. While developers have been using Unreal Engine 5 for simulations for some time, there was a lack of accessible, modernized examples of this workflow within the core repository.

Asset Creation in Blender

To upgrade the visual environments, I needed better models than the original low-fidelity simulation examples.

  • Optimized Models: Utilized Blender to model satellites and other rigid bodies from scratch, focusing on being loyal to the original simulation intent but with modern visual improvements and optimized polygon counts for real-time rendering.

Real-Time Visualization in Unreal Engine 5

With the 3D assets complete, the final step was integration.

  • Data Streaming: Configured the pipeline to stream live telemetry variables (such as position, rotation, and custom conditionals) directly from the Trick backend into Unreal Engine 5.
  • UE5 Optimizations: Configured and programmed the UE5 environment for a higher fidelity representation and visualization of the original simulations, ensuring smooth framerates and accurate physics synchronization.

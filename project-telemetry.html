Project Overview

Across all of my engineering projects—from autonomous robotics to specialized gas monitoring systems—reliable telemetry is the backbone that makes complex systems function. I consider robust sensor integration and real-time data visualization to be one of my core strengths. This project page details my methodology for bridging the gap between raw, low-level sensor voltages and high-level, actionable UI dashboards.

Hardware Acquisition & Sensor Control

Getting clean data starts at the hardware level. I frequently design custom pipelines that utilize microcontrollers to poll physical environments before handing that data off to a central compute unit.

  • Low-Level Polling: Utilized the Arduino Uno R3 as the primary data acquisition layer for the Autonomous Vehicle Challenge (AVC). The microcontroller handles the real-time polling of environmental sensors, ensuring the main computer's CPU isn't bogged down by constant I/O requests.
  • Precision ADC Processing: Engineered the logic to properly scale and map raw analog voltages. When working with 10->16 bit systems, I ensure precise analog-to-digital conversions, maintaining maximum fidelity for critical readings.
  • Pressure & Flow Control: Outside of robotics, I applied this same methodology to build real-time telemetry systems for monitoring industrial Alicat and PSI hardware, continuously validating pressure and flow states against safety tolerances.

Software Integration & AI Visualization

Raw numbers are useless if they cannot be interpreted quickly. I build custom software environments to fuse physical sensor data with advanced computer vision.

  • Computer Vision Pipeline: Deployed highly optimized YOLO v11 AI models on NVIDIA Jetson hardware to process live camera feeds. The system detects mission-critical objects (like colored balls and buckets for the AVC) and calculates their relative coordinates in real-time.
  • Unified UI/UX: Developed multithreaded desktop interfaces using PyQt6 and web-accessible dashboards using Flask. These GUIs plot the live sensor feeds streaming from a raspberry pi5.
  • Microcontroller SBC Communication: Programmed the pipelines between microcontrollers to single board computers utilizing a packet setup to make sure all sensor data after acquired reaches the computer at the same time. Implementing two way communication the whole way.

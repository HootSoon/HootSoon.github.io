Project Overview

During my NASA OSTEM Internship at the Johnson Space Center, my primary objective was to modernize the visual fidelity of aerospace example simulations. I achieved this by bridging NASA's core physics engine with industry-leading 3D rendering and modeling software.

The NASA Trick Framework

The Trick Simulation Environment is a powerful simulation architecture developed by NASA. It handles the heavy lifting of aerospace simulations—managing the rigorous physics calculations, math models, timing, and data logging required to simulate complex systems like orbital mechanics and rocket launches.

While Trick is incredibly robust at calculating the math, the default visual outputs for example simulations required an upgrade to help engineers and researchers better visualize and interact with the data in real-time.

Asset Creation in Blender

To upgrade the visual environments, I needed highly accurate 3D models to represent the telemetry data being calculated by Trick.

  • Hard Surface Modeling: Utilized Blender to model complex aerospace components from scratch, focusing on structural accuracy and optimized polygon counts for real-time rendering.
  • Historic Recreation: Modeled detailed representations of the Saturn V launch vehicle and the iconic F-1 rocket engines to be driven by the simulation data.

Real-Time Visualization in Unreal Engine 5

With the 3D assets complete, the final step was integration.

  • Data Streaming: Configured the pipeline to stream live telemetry variables (such as thrust, velocity, and spatial coordinates) directly from the Trick backend into Unreal Engine 5.
  • Dynamic Rendering: Programmed the UE5 environment to react dynamically to the Trick data, meaning the visual location, orientation, and engine states of the Blender models updated in perfect synchronization with the underlying physics math.
